# 进度跟踪

## 目标
帮助用户周期性检查学习进度，发现偏差并及时调整。**学了多少不等于掌握了多少——跟踪的是"掌握度"而非"完成度"。**

---

## 跟踪体系：三层频率

| 层级 | 频率 | 用时 | 核心问题 |
|------|------|------|---------|
| 日回顾 | 每晚 | 5-10 分钟 | 今天完成了什么？什么没完成？为什么？ |
| 周总结 | 每周日 | 20-30 分钟 | 本周各科进度百分比，与计划偏差多少？ |
| 月评估 | 每月底 | 30-60 分钟 | 整体进度是否需要调整下月计划？目标院校需要调整吗？ |

---

## 一、日回顾（5-10 分钟）

### 日回顾模板

```
📅 日期：____年__月__日  星期__

✅ 已完成：
- 数学：____（具体任务，如"近5年选择填空15题，正确率73%"）
- 408：____
- 英语：____
- 政治：____

❌ 未完成：
- ____（原因：时间不够 / 难度超预期 / 状态不好 / 临时有事）

📝 今日关键收获（最多写 1 条）：
- ____

🔧 明天调整：
- ____
```

> 日回顾的目的是"觉察"而非"追责"——没完成就标注原因，明天调整，不用自责。

## 三、月评估（30-60 分钟）

月评估 = 4 次周总结 + 1 次深度复盘。除了进度数字外，重点关注：

### 月度评估额外内容

1. **整体进度是否支持当前目标？**
   - 如果累计落后 > 15%：是否需要降低目标院校档次？
   - 如果连续 2 个月落后 > 10%：强烈建议重新评估目标

2. **各科时间分配是否合理？**
   - 统计本月各科实际投入时间 vs 计划投入时间
   - 数学/408 占比是否 ≥ 60%？（冲刺阶段应该 ≥ 65%）

3. **学习方法的有效性检查**
   - 刷了 200 道数学题但正确率没提升？→ 问题不是量不够，是方法不对（可能在刷已会的内容）
   - 背了 500 个单词但阅读正确率下降？→ 可能只是在机械背词，没有语境训练

4. **是否需要调整目标院校？**
   - 如果模考成绩与目标差距过大 → 触发动态调整模块（详见 mock-exam-adjustment.md）

## 四、偏差分析框架

当某科连续 2 周进度落后，按以下框架诊断：

```
问题：XX 科连续 X 周进度落后

1. 时间维度
   - 实际投入时间是否达到计划？
   - 如果没达到 → 原因是什么？（其他科挤占/效率低/动力不足）

2. 方法维度
   - 当前学习方法是否有效？（以正确率为检验标准）
   - 如果正确率停滞 → 是否需要换方法？（换老师？换资料？换刷题方式？）

3. 目标维度
   - 计划目标是否过高？（与同期平均水平对比）
   - 如果目标过高 → 降低本阶段目标，保证核心内容

4. 状态维度
   - 最近是否有影响状态的事件？（生病/家庭/感情/焦虑等）
   - 如果有 → 先解决问题，再恢复学习节奏
```

## 五、早期预警信号

以下信号出现任一，应触发深度检查：

| 🔴 预警信号 | 对应检查 |
|------------|---------|
| 某科连续 2 周进度偏差 > 15% | 时间分配 + 方法有效性 |
| 本周总学习时长 < 计划的 60% | 状态问题 + 外部干扰 |
| 某科正确率连续 2 周不升反降 | 方法问题 + 是否需要换资料 |
| 连续 3 天以上没做日回顾 | 动力问题 + 计划可执行性 |
| 频繁"今天状态不好明天补" | 可能是目标过高或倦怠期前兆 |
| 英语/政治挤占了数学/408 的时间 | 优先级错乱——数学和 408 永远是主战场 |

## 六、可视化管理建议

推荐用户建立一个简单的「学习仪表盘」，可以用：
- **Excel/飞书表格**：填写每周各科进度数字，自动生成趋势图
- **Notion**：建立考研 Dashboard，每个科目一个进度条
- **手写墙面表**：在书桌前贴一张大白纸，每天完成保底任务就画一个 ✔

推荐的仪表盘视图：
```

```

## 七、纠正措施库

根据偏差原因，选择对应的纠正措施：

| 偏差类型 | 纠正措施 |
|---------|---------|
| 时间不够 | 重新分配各科时间（数学/408 各 +30min，从英语/政治中扣）；或延长总学习时间 |
| 方法低效 | 换老师/换资料/换刷题方式；增加限时训练减少随意刷题 |
| 目标过高 | 降低本阶段目标（如"完成全部习题"→"完成重点章习题"） |
| 状态波动 | 检查作息/睡眠/运动情况；启用机动日休息半天 |
| 某科严重拖后腿 | 该科每天增加 1 小时专项突破；临时减少优势科投入 |
| 全面落后 | 触发目标重评估（详见 mock-exam-adjustment.md） |

## 输出格式

根据用户当前所处阶段，输出：
1. 日回顾模板（用户每天自己填）
2. 本周进度报告（如果用户提供了本周数据）
3. 月度评估报告（如果用户提供了月度数据）

## ⚠️ 关键原则
- 进度跟踪的目的是**发现问题和趋势**，不是给自己打分
- 某科进度领先不意味着可以减少投入——保持节奏
- 连续 2 周某科进度红灯 = 必须深挖原因，不能"下周再说"
- 建议用户在每周日晚上做周总结——养成习惯后 20 分钟就能完成
- 跟踪数据不需要完美——80% 准确就够了，不要花大量时间做"好看"的表格
