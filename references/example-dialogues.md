# 示例对话

以下场景展示不同用户状态下的回应思路和输出策略。

---

## 场景 1：迷茫新手 — 信息不全，Markdown 渐进引导

**用户**："我大三，普通二本计算机，想考研但完全不知道怎么开始。"

**回应思路**：
1. 先肯定（大三开始时间刚好）
2. 追问 2-3 个关键信息（"英语/数学基础怎样？有没有感兴趣的城市？"）
3. 给框架结论（本专业优势大，建议冲 211 稳双非保底）
4. 引导补充信息后出详细计划（"能告诉我更多细节的话，我可以出一份完整规划"）
5. **输出格式**：Markdown（信息不完整，不适合 HTML）

---

## 场景 2：目标明确 — 信息充分，触发 HTML 生成

**用户**："我是 2023 级福建农林大学软件工程的，想考厦大计算机专硕，刚学完高数，四级 520，专业排名前 10%，蓝桥杯省二，每天能学 6-8 小时。考 408 数二，有希望吗？怎么复习？"

**回应思路**：
1. 信息已足够完整 → 先 Markdown 给出结论亮点（"有希望！数二+本专业+竞赛是你的三张好牌"）
2. 综合评估：概率评估 + 择校三档 + 6 个月学习计划 + 每日作息
3. **同步准备 HTML 报告** — 信息覆盖了 5+ 维度且涉及全局规划
4. 告知用户："我可以帮你生成一份可保存、可打印的交互式 HTML 报告"
5. 收集完整 JSON → 运行 `generate_plan_html.py` → 交付 HTML

---

## 场景 3：模考焦虑 — 单一模块但需要详细分析

**用户**："我 10 月中模考，数一 72，408 考了 85，政治 55，英语 62，总分 274。目标武大 355。只剩两个月了，要换学校吗？"

**回应思路**：
1. 先安抚（"先别做任何重大决定，我们按步骤分析"）
2. 读取 mock-exam-adjustment.md → 出完整分析：
   - 成绩对标（274 vs 355，差距 81 分 🔴）
   - 单科短板（政治提分最快、数学保基础分、408 刷真题、英语稳住）
   - 总分推演（中性 333、乐观 357）
   - 三方案对比（坚持武大/换武理工/换地大）
3. 如果用户说"给我完整报告" → 触发 HTML
4. 如果只是要结论 → Markdown 就够

---

## 场景 4：报名季纠结 — 触发报名指导 + 动态调整联动

**用户**："10 月了，马上要正式报名。我模考总分 310，去年目标院校复试线 340。我要不要换学校？报名系统怎么填？"

**回应思路**：
1. 先出决策数据（差距 30 分 → 建议降档。参考 mock-exam-adjustment.md）
2. 再给报名操作指南（预报名的已经报了没？考点选对没？参考 registration-guide.md）
3. 提醒关键时间（10 月 25 日截止，截止后无法修改志愿）
4. **两个模块联动** → 适合 HTML（报名清单 + 模考分析 + 新院校计划 放在一个页面里）

---

## 场景 5：出分后 — 复试/调剂/二战 三岔路口

**用户**："出分了，总分 335，去年复试线 340，差 5 分。我现在是准备复试还是调剂还是直接二战？"

**回应思路**：
1. 判断：总分与复试线差 5 分 → 擦线，等复试线公布 + 同步准备
2. 读取 post-exam-decision.md → 给决策矩阵（复试 + 调剂双线并行，70% 精力复试 + 30% 调剂）
3. 立即行动清单（联系导师、对比近 3 年线、锁定调剂备选）
4. 如果用户说"我觉得没希望了想二战" → 引导先等复试线，不要提前放弃
5. **输出格式**：Markdown（信息敏感、需要快速响应，HTML 生成反而慢）
