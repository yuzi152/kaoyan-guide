## 项目简介

**考研规划指导**是一个基于 Claude Code SKILL 框架开发的智能考研助手技能。它不只是一个问答机器人——而是一个融合「规划师 + 学习教练 + 信息助手」三重角色的全流程决策与执行系统。

### 为什么做这个？

- 考研是高校学生最高频、最焦虑的场景之一
- 市面上的考研指导要么碎片化（论坛帖子），要么商业化（付费课程）
- 学生需要的不是零散信息，而是一个能**根据个人背景给出可执行计划**的智能体
- 每年约 400 万+ 考生中，很多人因为信息不对称和缺乏规划而失败

### 能做什么？

```
用户：我大三，双非软件工程，想考厦大计算机，有希望吗？
  ↓
SKILL：采集信息 → 多维度评估 → 择校三档推荐 → 6 个月学习计划 → 每日作息
  ↓
输出：Markdown 即时回复 + 可保存的交互式 HTML 报告（含雷达图、打卡系统）
```

### 设计理念：渐进式披露

```
上下文层级 1：description（197 字符）       → 始终在上下文，决定是否触发
上下文层级 2：SKILL.md 正文（155 行）       → SKILL 触发后加载
上下文层级 3：references/*.md（13 文件）    → 按需读取，匹配到哪个模块读哪个
上下文层级 4：scripts/*.py / assets/*.html  → 手动执行或调用时使用
```

## 快速开始

### 1. 安装技能

```bash
# 克隆到 Claude Code 技能目录
git clone <repo-url> ~/.claude/skills/kaoyan-guide

# 或手动复制
cp -r kaoyan-guide ~/.claude/skills/
```

### 2. 触发技能

在 Claude Code 对话中说任意考研相关话题：

```
"我想考研但不知道怎么开始"
"我是大三软件工程的，想考厦大计算机，帮我评估一下"
"模考数学 72 分，要不要换学校？"
"408 怎么刷题？"
"出分了差 5 分进复试怎么办"
```

### 3. 生成完整报告

```bash
# 步骤一：采集用户数据（交互式问卷）
python scripts/collect_user_info.py -o 我的数据.json

# 步骤二：生成 HTML 交互式报告
python scripts/generate_plan_html.py --data 我的数据.json --output 我的考研报告.html

# 步骤三：浏览器打开报告
# 报告包含：雷达图、进度条、倒计时、Tab 切换、每日打卡、打印/导出功能
```

## 脚本与资产

### scripts/collect_user_info.py

交互式命令行问卷，6 步采集用户完整信息：

```
① 基本信息   → 姓名、本科/目标院校专业、目标总分、考试日期
② 学习基础   → 英语/数学水平、每日可用时间、跨专业/在职/二战意向、GPA、竞赛
③ 各科进度   → 数学/408/英语/政治 当前进度% + 目标分数
④ 竞争力自评 → 6 维雷达图评分 (1-5) + 优劣势
⑤ 择校偏好   → 三档院校名称/层次/复试线
⑥ 作息偏好   → 使用默认模板或自定义
```

支持 `--quick` 快速模式、`--from-file` 续填模式。

### scripts/generate_plan_html.py

JSON → 交互式 HTML 报告生成器：

- SVG 雷达图（纯 SVG，无 CDN 依赖）
- CSS 进度条 + 响应式布局 + 打印样式
- JavaScript Tab 切换 + 倒计时 + localStorage 打卡
- 导出 Word / CSV 按钮

### assets/plan-template.html

独立 HTML 模板，内联所有 CSS + JS + SVG，零外部依赖。包含完整的设计系统变量（颜色、圆角、阴影）。

## 技术栈

| 层级 | 技术 |
|------|------|
| **框架** | Claude Code SKILL（YAML frontmatter + Markdown body + progressive disclosure） |

- 参考文档：纯 Markdown，任何编辑器可读

## 用户画像

技能内置 8 种用户画像的差异化策略：

| 画像 | 特点 | 专属策略 |
|------|------|---------|
| 本专业应届 | 基础好，时间充裕 | 冲高目标，注重效率 |
| 跨专业应届 | 专业课薄弱 | 提前启动专业课，目标务实 |
| 往届全职备考 | 时间充裕，心理压力大 | 严格执行计划，注意心理调节 |
| 往届在职备考 | 时间碎片化 | 高效利用晚上+周末，目标适度降低 |
| 二战考生 | 有经验，心理负担重 | 先做失败原因分析；目标需重新评估 |
| 专科考研 | 报考受限 | 确认同等学力政策；保底档选 B 区 |
| 大龄考研（25+） | 年龄焦虑 | 算清机会成本；非全为合理选项 |
| 保研边缘人 | 两手准备 | 明确保研失败节点；考研应急最晚 10 月启动 |

## 数据来源声明

本 SKILL 中引用的考研相关数据（复试线、报录比、招生名额等）来源包括：

- 中国研究生招生信息网（研招网，https://yz.chsi.com.cn）
- 各院校研究生院官网发布的招生简章与历年数据
- 教育部《全国硕士研究生招生工作管理规定》
- 常见考研辅导资料体系（肖秀荣系列、王道 408、张宇/汤家凤/武忠祥数学等）

> ⚠️ **声明**：以上数据在 SKILL 中作为参考框架使用，具体数值标注了"以官方最新公告为准"。技能不存储不传播未公开的招生数据，所有建议均引导用户自行到官方渠道核实。

所有 SKILL 逻辑、模块设计、输出模板、脚本代码均为小组原创。
